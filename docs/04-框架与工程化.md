## 第一阶段：LangChain — 确认层（Day 17-18）

### 1.1 为什么这时候才学框架

```
底层搞懂了 → 框架一看就明白

你手写的:  messages.append(tool_result) → call_llm(messages) → 解析
LangChain:  agent_executor.invoke({"input": query})
           └─ 内部就是帮你在做 messages 拼接 + 解析循环

你手写的:  Tool Schema 用 dict 描述
LangChain: @tool 装饰器，自动生成 Schema
           └─ 就是帮你在包 description + parameters JSON

学框架之前你已经知道它在做什么 → 不会被绕进去。
换框架也不怕 → 原理一样的。
```

### 1.2 LCEL — 管道思维

```python
"""
LCEL 管道: 和 Java Stream / Reactor 一样的思想

你手写的是: prompt → llm → parse → return
LCEL 写成:  prompt | llm | parser
"""
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

llm = ChatOpenAI(model="deepseek-chat", base_url="https://api.deepseek.com/v1")

# ===== RAG Chain — 一行管道 =====
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | ChatPromptTemplate.from_messages([
        ("system", "参考以下资料回答:\n{context}"),
        ("human", "{question}")
    ])
    | llm
    | StrOutputParser()
)

answer = rag_chain.invoke("100路末班车几点？")
print(answer)
```

### 1.3 LangGraph — 有状态的 Agent

```python
"""
LangGraph = 状态机引擎

你手写的 Agent 循环: while True: call_llm → parse → execute → append
LangGraph 写成:       StateGraph → add_node → add_edge → compile → invoke

底层原理一样，但 LangGraph 帮你处理了:
- 状态管理（messages 自动追加）
- 条件路由（tool_calls? → tools : END）
- 循环控制（max_iterations）
"""
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
from typing import TypedDict, Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]

graph = StateGraph(State)
graph.add_node("agent", lambda s: {"messages": [llm.bind_tools(tools).invoke(s["messages"])]})
graph.add_node("tools", ToolNode(tools))
graph.set_entry_point("agent")
graph.add_conditional_edges("agent", lambda s: "tools" if s["messages"][-1].tool_calls else END)
graph.add_edge("tools", "agent")

app = graph.compile()
result = app.invoke({"messages": [("user", "查西湖到火车东站公交")]})
print(result["messages"][-1].content)
```

### 2.1 上下文窗口管理 ⚠️ 最关键

> "这是大模型应用跟传统后端最不一样的地方。Agent 你得时刻盯着 Context Window 里还剩多少 Token。"

```python
"""
上下文管理策略 — 三种方案，不同场景用不同策略
"""

class ContextManager:
    """
    核心问题: 多轮对话后 messages 越来越长，超出 Context Window
    
    策略对比:
    ┌─────────────┬──────────────────┬──────────────────┐
    │ 策略         │ 做法              │ 适用场景          │
    ├─────────────┼──────────────────┼──────────────────┤
    │ 滑动窗口     │ 保留最近N轮        │ 客服对话（短期记忆）│
    │ 摘要压缩     │ LLM 压缩旧消息     │ 长任务（保留关键信息）│
    │ 选择性保留   │ 只保留 tool_calls  │ Agent 多步骤       │
    └─────────────┴──────────────────┴──────────────────┘
    """
    
    def __init__(self, max_tokens: int = 4000, model: str = "deepseek-chat"):
        self.max_tokens = max_tokens  # 给 messages 的最大 Token 预算
        self.model = model
        self.context_window = 128000  # DeepSeek 上下文窗口（留80%做输出）
    
    def estimate_tokens(self, messages: list) -> int:
        """快速估算 Token 数（约 2 chars/token 中文）"""
        total = sum(len(m["content"] or "") for m in messages if m.get("content"))
        # tool_calls 的 arguments 也要算
        for m in messages:
            for tc in m.get("tool_calls", []):
                total += len(tc.get("function", {}).get("arguments", ""))
        return total // 2  # 粗略估算
    
    def sliding_window(self, messages: list, keep_last: int = 10) -> list:
        """滑动窗口 — 只保留最近 N 条消息 + system prompt"""
        system = [m for m in messages if m["role"] == "system"]
        recent = messages[-keep_last:] if len(messages) > keep_last else messages
        return system + recent
    
    async def summarize_compress(self, messages: list, llm) -> list:
        """摘要压缩 — 用 LLM 把旧消息压成一条摘要"""
        if self.estimate_tokens(messages) < self.max_tokens:
            return messages
        
        # 前一半消息做摘要
        mid = len(messages) // 2
        old_msgs = messages[:mid]
        recent = messages[mid:]
        
        summary_prompt = "将以下对话历史压缩为一段摘要，保留关键信息：\n"
        summary_prompt += "\n".join([
            f"{m['role']}: {str(m.get('content',''))[:200]}"
            for m in old_msgs if m["role"] != "system"
        ])
        
        resp = await llm.ainvoke([{"role": "user", "content": summary_prompt}])
        
        return [messages[0]] + [  # System prompt
            {"role": "system", "content": f"历史摘要: {resp.content}"}
        ] + recent
```

### 2.2 Token 成本控制

```python
"""
Token 预算机制 — 文章说的"一个测试 Agent 挂了一晚上烧了二十多刀"

策略:
┌──────────────────┬──────────┬─────────────────────────┐
│ 策略              │ 节省比例  │ 实现方式                 │
├──────────────────┼──────────┼─────────────────────────┤
│ 缓存相同问题       │ 30-50%   │ Redis + 语义相似度匹配    │
│ 小模型预筛选       │ 20-30%   │ 简单意图用 cheap model   │
│ 提示压缩           │ 15-25%   │ 去除冗余 System Prompt   │
│ 输出长度限制       │ 10-20%   │ max_tokens 限制          │
└──────────────────┴──────────┴─────────────────────────┘
"""

class TokenBudget:
    """Token 预算管理器"""
    
    def __init__(self, daily_limit: float = 5.0):
        self.daily_limit = daily_limit  # 每日预算(美元)
        self.used_today = 0.0
        self.cache = {}  # 简单内存缓存（生产用 Redis）
    
    def estimate_cost(self, prompt_tokens: int, completion_tokens: int, model: str) -> float:
        """估算成本"""
        prices = {
            "deepseek-chat": (0.14, 0.28),       # 输入/输出 $/1M tokens
            "gpt-4": (30, 60),
            "gpt-3.5-turbo": (0.5, 1.5),
        }
        in_price, out_price = prices.get(model, (1, 2))
        return (prompt_tokens / 1_000_000) * in_price + (completion_tokens / 1_000_000) * out_price
    
    def check_budget(self, estimated_cost: float) -> bool:
        """检查是否超出预算"""
        if self.used_today + estimated_cost > self.daily_limit:
            raise BudgetExceededError(f"超出每日预算 ${self.daily_limit}")
        return True
    
    def get_from_cache(self, query: str) -> str | None:
        """缓存的 key 是按语义相似度，不是精确匹配"""
        for cached_q, cached_a in self.cache.items():
            if self._similar(query, cached_q):
                return cached_a
        return None
    
    def _similar(self, a: str, b: str) -> bool:
        """简单相似度判断（生产用向量相似度）"""
        return len(set(a) & set(b)) / max(len(set(a) | set(b)), 1) > 0.8
```

### 2.3 可观测性

```python
"""
Agent 可观测性 — 三大支柱

面试官问"监控怎么做的"时，你能答:
1. 日志: 每个 Agent 步骤的结构化日志
2. 指标: Token 用量、响应延迟、成功率、缓存命中率
3. 追踪: 工具调用链跟踪
"""

import time
import logging
from dataclasses import dataclass, field
from typing import Any

logger = logging.getLogger("agent")

@dataclass
class AgentMetrics:
    """Agent 运行指标"""
    total_calls: int = 0
    total_tokens: int = 0
    total_cost: float = 0.0
    avg_latency_ms: float = 0.0
    tool_call_success_rate: float = 0.0
    cache_hit_rate: float = 0.0

class ObservableAgent:
    """可观测 Agent — 每个关键节点埋点"""
    
    def __init__(self):
        self.metrics = AgentMetrics()
    
    async def run_with_tracing(self, query: str) -> dict:
        step_start = time.time()
        
        # 1. LLM 调用埋点
        llm_start = time.time()
        llm_resp = await self.call_llm(query)
        llm_ms = (time.time() - llm_start) * 1000
        
        tokens = llm_resp.get("usage", {}).get("total_tokens", 0)
        logger.info("agent.llm_call", extra={
            "latency_ms": llm_ms,
            "tokens": tokens,
            "model": "deepseek-chat"
        })
        
        # 2. 工具调用埋点
        if llm_resp.get("tool_calls"):
            tool_start = time.time()
            tool_result = await self.execute_tools(llm_resp["tool_calls"])
            tool_ms = (time.time() - tool_start) * 1000
            logger.info("agent.tool_call", extra={
                "tool": llm_resp["tool_calls"][0]["function"]["name"],
                "latency_ms": tool_ms,
                "success": tool_result.get("success", False)
            })
        
        # 3. 更新指标
        self.metrics.total_calls += 1
        self.metrics.total_tokens += tokens
        total_ms = (time.time() - step_start) * 1000
        
        return {
            "response": llm_resp["choices"][0]["message"]["content"],
            "trace": {
                "llm_latency_ms": llm_ms,
                "total_latency_ms": total_ms,
                "tokens_used": tokens,
            }
        }
```

### 2.4 降级与容错

```python
"""
降级策略 — LLM 挂了怎么办？

Plan A: 主模型（GPT-4/DeepSeek）
  ↓ 超时/错误
Plan B: 备用模型（GPT-3.5-turbo）
  ↓ 仍然失败
Plan C: 规则兜底（返回预设话术）
"""

FALLBACK_RESPONSES = {
    "route_query": "抱歉，当前无法查询路线，请拨打客服热线 0571-xxxx。",
    "complaint": "您的投诉已记录，系统恢复后第一时间处理。工单号将在恢复后短信通知。",
    "default": "服务暂时不可用，请稍后重试或联系客服。",
}

async def call_with_fallback(messages: list, intent: str = "default") -> str:
    """带降级的 LLM 调用"""
    try:
        return await call_primary_model(messages)
    except (TimeoutError, ConnectionError):
        logger.warning("主模型不可用，降级到备用")
        try:
            return await call_backup_model(messages)
        except Exception:
            logger.error("所有模型不可用，使用规则兜底")
            return FALLBACK_RESPONSES.get(intent, FALLBACK_RESPONSES["default"])
```
