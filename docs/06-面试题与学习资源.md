## 高频面试题

### 基础概念

**Q1: Token 是什么？一段 1000 字的中文文本大约消耗多少 Token？**

> 约 1500-2000 Token。中文每个字约 1.5-2 Token，不等同于"字"。Token 是 BPE/WordPiece 分词后的最小单元。

**Q2: Temperature 0 和 1.0 有什么区别？实际项目怎么选？**

> 0 = 确定性输出（代码生成、数据提取用 0-0.3）；1 = 高随机性（创意写作）。Agent 项目通常 0.3-0.5——既要稳定选工具又要一定灵活性。

**Q3: RAG 和 Fine-tuning 怎么选？**

```
数据频繁变化？ → RAG（更新文档即可）
需要改变模型"说话方式"？ → Fine-tuning
有足够标注数据吗？ → Fine-tuning，没有 → RAG
预算有限？ → RAG（$0.001/次 vs 训练一次 $50-200）
```

**Q4: 什么是 Agent？和传统 Chatbot 有什么区别？**

> Agent = LLM + Planning + Memory + Tool Use。Chatbot 被动响应单轮，Agent 主动规划多步、自主选工具、循环执行。

**Q5: Function Calling 的消息格式长什么样？工具执行完后消息怎么传？**

> assistant 消息带 `tool_calls` 数组（content=null），然后追加一条 role=tool 的消息（带 tool_call_id 匹配）。这两条必须成对出现。

### Agent 架构

**Q6: ReAct 和 Plan-and-Execute 有什么区别？什么时候用哪个？**

| | ReAct | Plan-and-Execute |
|---|---|---|
| 决策 | 每步决策 | 全局规划+分步执行 |
| 长任务 | 容易丢失目标 | 保持全局视角 |
| 适用 | 开放式探索 | 结构化多步骤 |

**Q7: Multi-Agent 系统中怎么处理通信和冲突？**

> 消息队列（发布/订阅）、共享状态（LangGraph State）、层级裁决（Supervisor Agent 最终决策）。

**Q8: Agent 记忆系统怎么设计？**

> 三层：工作记忆（当前消息列表）→ 短期记忆（滑动窗口+摘要压缩）→ 长期记忆（向量库+关系库持久化）。

### 工程实践

**Q9: Token 超出 Context Window 你怎么处理？**

> 三种策略：滑动窗口（保留最近N轮）适合客服；摘要压缩（LLM压旧消息）适合长任务；选择性保留（只留 tool_calls 结果）适合多步 Agent。

**Q10: 工具执行失败怎么兜底？**

> 重试（指数退避）→ 空结果兜底（明确告知）→ 降级（换备用工具）→ 最终规则兜底。

**Q11: 生产环境 Agent 的可观测性怎么做？**

> 日志（结构化每步日志）、指标（Token用量/延迟/成功率/缓存命中率）、追踪（OpenTelemetry 链路）。

**Q12: RAG 召回准确率低你怎么排查？**

> 四个方向：Chunk Size 不合适（跑实验对比）、Embedding 模型不匹配（换 BGE-M3 测）、召回后没重排序（加 Reranker）、结构化数据没混合检索（SQL+向量）。

**Q13: 你上过线吗？碰到过什么工程问题？**

> 结合你的终极项目回答：Agent 循环死循环（加 max_iterations + 循环检测）、幻觉（加 RAG 约束 + 关键信息后校验）、并发冲突（分布式锁）、成本失控（缓存 + 小模型预筛选 + Token 预算）。
